사용자가 `/docs`를 실행하면 `docs/` 디렉토리의 모든 문서를 읽고 프로젝트 컨텍스트를 파악한다.

## 실행 절차

1. `docs/` 디렉토리가 존재하는지 확인한다.
   - 없으면 사용자에게 `/makedocs`로 먼저 문서를 생성하라고 안내한다.

2. `docs/` 안의 모든 `.md` 파일을 읽는다.
   - docs/README.md 부터 시작
   - architecture.md, local-dev.md, deployment.md, cicd.md 순으로

3. 읽은 내용을 바탕으로 다음을 간략히 요약해서 사용자에게 보고한다:
   - 프로젝트가 무엇인지 (한 줄)
   - 기술 스택 핵심 (한 줄)
   - 로컬 실행 방법 (핵심만)
   - 배포 환경 (있는 경우, 핵심만)
   - 특이사항이나 주의할 점

4. "프로젝트 파악 완료. 작업 지시해줘!" 로 마무리한다.

## 원칙

- 문서를 읽는 것으로 끝내지 않고, 반드시 요약을 출력해 사용자가 파악 여부를 확인할 수 있게 한다
- 문서에 없는 내용은 추측하지 않는다
- 문서가 오래됐거나 누락된 정보가 있으면 `/makedocs`로 갱신을 권장한다
